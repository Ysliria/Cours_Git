Les bases

Initialisation d'un projet

Maintenant que Git est configuré, opn peut initialisé un projet avec git init. Pour cela, créé un dossier vide et allez vous placer dedans en ligne de commande. 


    λ git init
    Initialized empty Git repository in E:/Développements/Cours_PHP/Cours_Git/.git/

Connaitre l'état du projet

Pour connnaître l'état d'un projet, il existe la commande git status. Créez des dossiers et/ou fichiers dans votre projet et essayez. 


    λ git status
    On branch master

    No commits yet

    Untracked files:
    (use "git add <file>..." to include in what will be committed)
    cours/
    scripts/
    styles/

    nothing added to commit but untracked files present (use "git add" to track)

Il nous indique la liste des fichiers du projet qui ne sont pas suivi par Git, et nous propose de les ajouter.

Ajouter des fichiers au suivi de Git

Pour ajouter des fichiers à la surveillance de Git la commande est git add. Avec cette commande vous devez préciser ce que vous souhaitez ajouter à l'index de git. Si vous ajoutez un dossier il ajoutera tout son contenu aussi. Si c'est un fichier, il n'ajoutera que le fichier nommé. Maisis cous souhaitez tout ajouter, alors on utilisera le Joker *.

git add cours/* puis git status pour le résultat 

    λ git add cours/*

    λ git status
    On branch master

    No commits yet

    Changes to be committed:
      (use "git rm --cached <file>..." to unstage)
            new file:   cours/1_introduction.html
            new file:   cours/2_bases.html

    Untracked files:
      (use "git add <file>..." to include in what will be committed)
            scripts/
            styles/
git add styles/ puis git status pour le résultat 

    λ git add styles/

    λ git status
    On branch master

    No commits yet

    Changes to be committed:
      (use "git rm --cached <file>..." to unstage)
            new file:   cours/1_introduction.html
            new file:   cours/2_bases.html
            new file:   styles/prism.css

    Untracked files:
      (use "git add <file>..." to include in what will be committed)
            scripts/
git add * puis git status pour le résultat 

    λ git add *

    λ git status
    On branch master

    No commits yet

    Changes to be committed:
      (use "git rm --cached <file>..." to unstage)
            new file:   cours/1_introduction.html
            new file:   cours/2_bases.html
            new file:   scripts/prism.js
            new file:   styles/prism.css

Faire un premier commit

Une fois les fichiers trackés par Git, ils ne sont pas pour autant enregistrés. Pour cela il faut faire ce qu'on appel un commit. Le commit permet l'enregistrement des modifications dans le dépôt Git. Attention Git étant décentralisé, quand on parle de dépôts on est, ici, toujours en local.
La commande s'appèle git commit, on lui ajoute le commutateur -m pour ajouter un commentaire. 


    λ git commit -m "Initilisation du projet"
    [master (root-commit) e228b02] Initilisation du projet
    4 files changed, 495 insertions(+)
    create mode 100644 cours/1_introduction.html
    create mode 100644 cours/2_bases.html
    create mode 100644 scripts/prism.js
    create mode 100644 styles/prism.css
