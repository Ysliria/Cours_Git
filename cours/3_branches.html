Les branches

Qu'est-ce qu'une branches

La gestion de sources avec Git ne serait rien sans son concept de branches. Si on parle "d'arbre Git", ce n'est pas pour rien.
Une branche est forme de copie du tronc (appelé aussi branche principale, main ou master) qui permet d'effectuer des modifications du projet sans impacter le projet existant, mais en partant de la même base. Elle se liste avec git branch 


    λ git branch
    * master

Ici nous n'avons, pour l'instant, que la branche principale. La petit étoile devant nous indique sur laquelle on travaille.

Créer une branche

Pour créer une branche, on utilise la commande git branch nomDeLaBranche. Créons un branche development. 


    λ git branch development

    λ git branch
    development
    * master

Je peux aussi faire de l'arborescence de dans mes branches : 


    λ git branch feature/issu_1

    λ git branch
    development
    feature/issu_1
    * master

Changer de branche

Maintenant que ma branche est créé il faut que je me place dessus pour pouvoir travailler sur celle-ci. Pour cela on doit faire un git checkout avec le nom de la branche. 


    λ git checkout development
    Switched to branch 'development'
    M       cours/2_bases.html
    A       cours/3_branches.html

Une fois sur notre branche de travail, on peut faire les commits que l'on souhaite sans impacter la branche principale.

Mettre à jour notre branche principale

On travaille sur notre branche "development" mais il nous faut bien apporter nos modifications à notre branche principale pour en faire bénéficier tout le projet. Et pour ce faire on utilisera la commande git merge avec le nom de la branche à merger. Et avant cela il faut se remettre sur la branche sur laquelle on veut merger, à savoir pour nous master.
