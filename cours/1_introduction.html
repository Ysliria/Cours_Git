Présentation

Qu'est-ce que Git ?

Git est un système de gestion de vesions décentalisé créé par Linus Torvalds. Ce qui signifie qu'il va géré les versions d'un projet (logiciel, document, etc ...), pas sur un serveur centralisé comme le ferait SVN, mais sur toutes les machines ayant une copie des sources.

Git est une véritable machine à remonter le temps, permettant de revenir en arrière dans les sources du projet et donc d'en voir plusieurs versions. Il a aussi pour but de facilité le travaille en équipe, mais aussi sur des projets personnels. Il est indispensable au travail du développeur.

Un peu d'histoire

Linus Torvalds etant le seul à gérer les sources de Linux, il se retrouve surchargé de travail du fait de la quantité grandissante des contributeurs. Il décide donc, en 2002, de migrer sur BitKeeper de Larry McVoy, un VCS gratuit pour les projets open source. Mais voilà, la version gratuite ne permettait pas gérer les métadonnées. Alors Andrew Tridgell, contributeur du noyau de Linux et créateur de Samba, décide de créer un système pour récupérer ses métadonnées. Ceci sera très mal vu par Larry McVoy, qui estime qu'on doit jouer le jeu des limitations quand on prend une version gratuite. En février 2005, il décide donc d'arrêter la version gratuite de BitKeeper à partir de juillet 2005.

La situation mettant sous tension le projet Linux, Linus Torvalds décide de se pencher sur la question, en palliant au passage les inconvennients qu'il avait rencontrés. Le 20 avril 2005 Linus Torvalds annonce la sortie de Git !

Installation

Pour installer Git sous Windows, rien de bien compliqué ! Allez sur le site de Git et télécharger la dernière version. Ensuite, laissez-vous guider par l'installeur. Ensuite pour valider l'installation saisissez git --version pour vérifier qu'il est bien installé et que vous êtes sous la bonne version. 


    λ git --version
    git version 2.31.1.windows.1

Félicitation ! Git est installé et prêt à fonctionner.

Configuration

Comme tout outils, Git doit être configuré. Pour se faire on va commencer par configurer notre adresse mail et notre nom. 


    λ git config --global user.name "Mickaël AUGER"
    λ git config --global user.email auger.mickael37@gmail.com

En utilisant le commutateur --global on s'assure que la configuration sera faite pour tous les projets. Si on fait un git congif --list on aura la liste des configurations enregistrées pour le global. Avec le commutateur --show-origin on ajoute l'emplacement des différentes configurations.
